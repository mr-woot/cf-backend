ASSIGNMENT – CAPITAL FLOAT

  • Music tracks listing web app that also keeps record of the search queries.

  • Tech Stack Used:

    • NodeJs
    • AngularJs
    • ExpressJs
    • MongoDb

  • Architecture (2-tier):

    • Backend uses NodeJs, ExpressJs.
    • Frontend uses AngularJs, Bower, HTML, CSS.
    • 3rd Party API used: Last.fm

  • Use Cases:

    • Home Page lists top tracks worldwide (3rd Party Integration).
    • Search Page finds matching tracks according to the track name (3rd Party Integration).
    • Search Page simultaneously also saves the search history in the custom backend using the API created.

  • Additional features:

    • Delete all history (API functionality only).
    • Delete specific searched key (API functionality only).
    • 3rd Party API integrated in custom backend, so as to have modularity in code.
    • Recently searched queries (Always displays unique search queries).

  • Internal Flow:

    • User requests for e.g. ‘/tracks’, frontend fires API request to the custom backend, which in turn sends the response collected from the 3rd Party API response.
    • For search history, query searched is send via POST request to the custom backend which validates the query and in turn saves the query in the mongo database which is hosted on mLab platform via Heroku.

  • API Endpoints:

    • Frontend:

      • ‘/’: Displays top tracks worldwide.
      • ‘/search’: Searches for the given track.

    • Backend:

      • ‘GET /tracks’: Fetches top tracks worldwide.
      • ‘GET /searchTrack?track={query}’: Searches for a query.
      • ‘GET /searchHistory’: Get the saved search history.
      • ‘POST /searchHistory’: Saves the recently searched track to the db.
      • ‘DELETE /searchHistory’: Deletes all the searched tracks in the db.
      • ‘DELETE /searchHistory/:id’: Deletes specific track from db, where id being Mongo ObjectID.
